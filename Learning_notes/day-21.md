# Why Track Tokens?

Every request to an LLM consumes tokens.

```text
Prompt
    ↓
Input Tokens

LLM

Response
    ↓
Output Tokens
```

Providers charge based on token usage.

---

# Tokens

Tokens are small pieces of text processed by the model.

Example:

```text
Hello world
```

may become:

```text
Hello
world
```

Two tokens.

---

# Input Tokens

Tokens sent to the model.

Example:

```text
What is AI?
```

Input tokens include:

* System prompt
* Summary memory
* Conversation history
* Current question

---

# Output Tokens

Tokens generated by the model.

Example:

```text
Artificial Intelligence is the simulation of human intelligence by machines.
```

These are output tokens.

---

# Total Tokens

```text
Total Tokens

=

Input Tokens

+

Output Tokens
```

---

# Session Metrics

Created in:

```python
metrics.py
```

Variables:

```python
total_input_tokens = 0

total_output_tokens = 0

total_cost = 0
```

Purpose:

Store cumulative statistics for the entire session.

---

# Estimating Tokens

Function:

```python
def estimate_tokens(
    text
):

    return len(
        text
    ) // 4
```

Approximation:

```text
1 token ≈ 4 characters
```

Example:

```python
estimate_tokens(
    "Hello world"
)
```

returns:

```python
2
```

---

# Pricing

```python
INPUT_COST_PER_1K = 0.0001

OUTPUT_COST_PER_1K = 0.0004
```

Meaning:

Cost per thousand tokens.

---

# Calculating Cost

Function:

```python
def calculate_cost(
    input_tokens,
    output_tokens
)
```

Formula:

```text
Input Cost

+

Output Cost
```

---

# Input Cost

```python
input_cost = (
    input_tokens / 1000
) * INPUT_COST_PER_1K
```

---

# Output Cost

```python
output_cost = (
    output_tokens / 1000
) * OUTPUT_COST_PER_1K
```

---

# Total Cost

```python
return (
    input_cost
    +
    output_cost
)
```

---

# Updating Session Totals

Function:

```python
update_metrics(
    input_tokens,
    output_tokens,
    cost
)
```

Purpose:

Accumulate metrics over multiple requests.

---

# Why Use Global Variables?

Variables:

```python
total_input_tokens

total_output_tokens

total_cost
```

exist outside the function.

Inside:

```python
update_metrics()
```

we modify them using:

```python
global total_input_tokens

global total_output_tokens

global total_cost
```

Purpose:

Update the original variables instead of creating local variables.

---

# Example

Initially:

```python
total_input_tokens = 0
```

Request 1:

```python
update_metrics(
    100,
    50,
    0.001
)
```

Results:

```python
total_input_tokens = 100

total_output_tokens = 50

total_cost = 0.001
```

Request 2:

```python
update_metrics(
    80,
    40,
    0.0008
)
```

Results:

```python
total_input_tokens = 180

total_output_tokens = 90

total_cost = 0.0018
```

---

# Why Use update_metrics()?

Without it:

Each request would overwrite previous values.

With it:

Statistics accumulate throughout the session.

# Showing Usage

Command:

```text
/usage
```

Purpose:

Display current statistics.

Example:

```text
Input Tokens : 2812

Output Tokens: 394

Total Tokens : 3206

Total Cost   : $0.000439
```

---

# Meaning of

```python
f"Total Cost : ${total_cost:.6f}"
```

Breakdown:

```python
f"..."
```

creates an f-string.

```python
{total_cost}
```

inserts the variable value.

```python
:.6f
```

means:

Display six digits after the decimal point.

Example:

```python
0.000439126
```

becomes:

```text
0.000439
```

---

# Why Input Tokens Are Higher

Input tokens contain:

```text
System Prompt
+
Summary Memory
+
Conversation History
+
Current Question
```

Example:

```text
500

+

1000

+

1200

+

20

=

2720 input tokens
```

Response:

```text
Artificial Intelligence is...
```

may contain:

```text
394 output tokens
```

Large input token counts are normal.

---

# More Accurate Measurement

Instead of:

```python
estimate_tokens(
    request.prompt
)
```

use:

```python
estimate_tokens(
    full_prompt
)
```

because Gemini receives:

```python
full_prompt
```

which contains:

```text
System Prompt

+

Conversation Context
```

# Key Takeaways

* Tokens determine LLM cost.
* Input tokens are what we send.
* Output tokens are what the model generates.
* Cost depends on both input and output tokens.
* Session metrics accumulate across requests.
* Global variables allow updating shared totals.
* `/usage` provides runtime statistics.
* Input tokens are usually larger because memory and system prompts are included.
* Tracking usage is essential in production AI systems.
