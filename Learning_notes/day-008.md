# Day 008 - Python Project Structure and Modules

## Objective

Learn how professional Python projects are organized using modules and a structured project layout.

---

## Topics Covered

### What is a Module?

A module is a Python file (`.py`) that contains code such as functions, classes, or variables.

Example:

```python
# llm.py

def ask_llm():
    pass
```

The module can be imported into another file:

```python
from llm import ask_llm
```

This allows code to be reused across multiple files.

---

### Why Use Modules?

Without modules:

```text
app.py
├── 500+ lines of code
```

Problems:

* Difficult to maintain
* Hard to debug
* Difficult to reuse code

With modules:

```text
main.py
llm.py
config.py
```

Benefits:

* Better organization
* Easier maintenance
* Improved readability
* Reusable components

---

### Imports

Imports allow one Python file to use code from another Python file.

Example:

```python
from llm import ask_llm
```

Python loads the required function from the specified module.

---

### Project Structure

A project structure is the organization of files and folders within a project.

Example:

```text
project/
├── src/
├── tests/
├── learning_notes/
├── README.md
├── .gitignore
```

A good structure makes projects easier to understand and maintain.

---

### What is the src Layout?

The `src` folder contains the application's source code.

Example:

```text
src/
├── main.py
├── llm.py
├── config.py
```

Purpose:

* Separates source code from documentation.
* Keeps projects organized.
* Commonly used in professional Python projects.

---

### Separation of Concerns

Each file should have one responsibility.

Example:

```text
config.py
```

Responsible for:

* Environment variables
* API keys
* Configuration settings

---

```text
llm.py
```

Responsible for:

* Communicating with Gemini
* Sending prompts
* Receiving responses

---

```text
main.py
```

Responsible for:

* User input
* Program execution
* Displaying output

---
