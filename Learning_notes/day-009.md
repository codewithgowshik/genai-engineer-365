# Day 009 - Dependency Management and Configuration Module

## Objective

Learn how Python projects manage external packages and how configuration values are separated from application logic using a configuration module.

---

## Topics Covered

### What are Dependencies?

Dependencies are external packages or libraries that a project requires to function.

Examples:

* `google-genai`
* `python-dotenv`
* `rich`

These packages are installed using `pip`.

Example:

```bash
pip install google-genai
```

---

### Why Dependency Management is Important

Without dependency management:

* Team members may not know which packages are required.
* Projects become difficult to reproduce.
* Different package versions can cause errors.

Dependency management ensures that everyone uses the same packages and versions.

---

### requirements.txt

The `requirements.txt` file contains the list of packages required by a project.

Example:

```text
google-genai
python-dotenv
rich
```

Other developers can install all dependencies using:

```bash
pip install -r requirements.txt
```

This makes projects portable and reproducible.

---

### pip freeze

Python can automatically generate a list of installed packages.

Example:

```bash
pip freeze > requirements.txt
```

This command stores all installed packages and their versions inside the requirements file.

Example:

```text
google-genai==1.0.0
python-dotenv==1.1.0
rich==14.0.0
```

---

### Virtual Environment

A virtual environment creates an isolated workspace for a project.

Benefits:

* Prevents package conflicts.
* Keeps dependencies separated between projects.
* Allows different projects to use different package versions.

Example:

```bash
python -m venv .venv
```

---

### Configuration Module

Configuration values should not be scattered throughout the project.

Instead, they should be stored in a dedicated module.

Example:

```python
# config.py

import os
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("API_KEY")

MODEL = "gemini-2.5-flash"
```

This centralizes configuration settings.

---

### Separation of Concerns

Professional projects separate responsibilities across different modules.

Example:

#### config.py

Responsible for:

* Environment variables
* API keys
* Configuration values

---

#### llm.py

Responsible for:

* Communicating with Gemini
* Sending prompts
* Receiving responses

---

#### main.py

Responsible for:

* User interaction
* Running the application
* Displaying output

---

```text
src/
└── day-009/
    ├── config.py
    ├── llm.py
    └── main.py
```

This structure improves readability and maintainability.
