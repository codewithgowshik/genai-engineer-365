# What is an Assistant Loop?

The assistant loop is the main cycle of the chatbot.

```text
User Input
      ↓

Process Command?
      ↓

Call LLM
      ↓

Generate Response
      ↓

Update Memory
      ↓

Save Conversation
      ↓

Wait For Next Input
```

This cycle repeats until the user exits.

# CLI UX Best Practices

Good command-line applications should:

### Clear Prompt

```text
Assistant >
```

instead of:

```text
>
```

---

### Handle Empty Input

```python
if not question.strip():
    continue
```

Purpose:

Ignore blank messages.

---

### Graceful Exit

```text
/exit
```

should stop the application cleanly.

# Virtual Environment

Created using:

```bash
python -m venv .venv
```

Purpose:

Isolate project dependencies.

---

# Why Source Code Stays Outside .venv

Correct:

```text
project/

├── .venv/
└── src/
```

Wrong:

```text
project/

└── .venv/
    └── src/
```

Rule:

```text
.venv belongs to Python

src belongs to you
```

---

# requirements.txt

Stores project dependencies.

Example:

```text
google-genai
pydantic
python-dotenv
rich
typer
```

Install:

```bash
pip install -r requirements.txt
```

---

# .gitignore

Purpose:

Prevent unnecessary files from being uploaded.

Example:

```text
.venv/
.env
__pycache__/
*.pyc
conversation.json
```

---

# Why Ignore .env

Contains secrets:

```text
GEMINI_API_KEY=...
```

Never upload API keys.

---

# Why Ignore .venv

Contains:

```text
Installed packages
Python interpreter
Libraries
```

Can be recreated using:

```bash
pip install -r requirements.txt
```

---

# Why Ignore **pycache**

Automatically generated by Python.

Can be recreated at any time.
